# 🚀 개발 및 테스트 가이드

## Mock 데이터로 개발/테스트하기

현재 백엔드가 없어도 프론트엔드를 테스트할 수 있도록 **Mock 데이터**가 구현되어 있습니다.

## 로그인 방법

### 방법 1: 아무 이메일/비밀번호로 로그인
로그인 폼에 **아무 값이나** 입력하면 됩니다.

```
이메일: test@test.com (또는 아무거나)
비밀번호: 12345678 (또는 아무거나)
```

**로그인 버튼**을 클릭하면 자동으로 테스트 계정으로 로그인됩니다.

### 방법 2: 회원가입 후 로그인
1. 회원가입 페이지에서 정보 입력
2. 회원가입 완료 (실제 저장은 안 됨)
3. 로그인 페이지로 이동
4. 아무 값이나 입력하고 로그인

## 테스트 시나리오

### ✅ 시나리오 1: 로그인 → 피드 확인
1. `/login` 페이지 접속
2. 아무 이메일/비밀번호 입력
3. 로그인 성공 → `/feed`로 이동
4. 4개의 게시글 확인

### ✅ 시나리오 2: 게시글 좋아요/북마크
1. 피드에서 게시글 하트 아이콘 클릭
2. 좋아요 수 증가/감소 확인
3. 북마크 아이콘 클릭
4. 북마크 상태 변경 확인

### ✅ 시나리오 3: 유료 컨텐츠 구매 (UI만)
1. 유료 게시글 (9,900P) 클릭
2. 잠금 상태 확인
3. "구매하기" 버튼 (미구현이지만 UI는 표시됨)

### ✅ 시나리오 4: 통계 확인
1. 상단 메뉴에서 "통계" 클릭
2. 대시보드 차트 확인
3. 조회수/매출 그래프 확인
4. 인기 게시글 Top 5 확인

### ✅ 시나리오 5: 사이드바 확인
1. 프로필 정보 (이름, 팔로워/팔로잉/게시글 수)
2. 포인트 잔액: 15,000P
3. 메뉴 네비게이션

## 백엔드 연동 준비

Mock 데이터는 다음 위치에 구현되어 있습니다:

### 1. LoginForm.jsx
```javascript

const response = await authService.login(formData)
login(response.data.user, response.data.token)
```

### 2. FeedPage.jsx
```javascript

setPosts(response.data)
```

### 3. StatisticsPage.jsx
```javascript
// TODO 주석 아래 코드 주석 해제

setStats(response.data)
```

백엔드 API가 준비되면:
1. Mock 데이터 부분 주석 처리

3. Spring Boot 서버 실행 (http://localhost:8080)
